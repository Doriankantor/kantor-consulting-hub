--------------------------------------------------------------------------------
1. CORE RULE
--------------------------------------------------------------------------------
EVERY NewsAPI query MUST combine:
   (1) a drone / UAS / UAV term   AND
   (2) a Latin-American country, region, or armed-actor term

No query ever searches a drone term alone or a place term alone — always both,
joined with AND. Multi-word or hyphenated terms are wrapped in quotes so NewsAPI
matches them as an exact phrase.

Examples of the actual query string sent to NewsAPI:
   drone AND Colombia
   drone AND "El Salvador"
   FARC AND dron
   drone AND military AND Colombia
   compra AND dron AND militar AND México
   drone AND "fuerzas armadas" AND Brasil
   "counter-drone" AND "Latin America"
   "C-UAS" AND Colombia
   Iranian AND drone AND "Latin America"


--------------------------------------------------------------------------------
2. NEWSAPI FETCH PARAMETERS (per query call)
--------------------------------------------------------------------------------
Endpoint     : https://newsapi.org/v2/everything
q            : strict AND query (drone term AND place/actor term)
language     : "en" or "es" (set per query — never both for the same query)
from         : now minus 72 hours        (LOOKBACK_HOURS = 72)
sortBy       : publishedAt
pageSize     : 100                        (PAGE_SIZE = 100)
max pages    : 2                          (MAX_PAGES = 2; breaks early if < 100)
total queries: 70 separate API calls per run

WHY 72h (not 24h):
  NewsAPI's free Developer plan serves articles with a ~24h delay, so a strict
  "last 24h" window returns nothing. 72h overlaps the delayed-availability zone
  and recovers a missed run. URL de-duplication prevents re-inserting anything
  already stored, so the wider window never creates duplicates.

QUOTA NOTE:
  Free Developer plan = 100 requests / 24h. One full run uses 70. Fine for the
  once-daily schedule; allows roughly one extra manual run per day.

--------------------------------------------------------------------------------
4. DE-DUPLICATION
--------------------------------------------------------------------------------
  (a) Within a run  — collapse fetched articles by `url`.
  (b) Against the DB — query existing cs_articles.url (chunked, 100 at a time)
                       and drop any URL already stored.
  Result: a URL is never inserted twice.


--------------------------------------------------------------------------------
5. CLAUDE RELEVANCE GATE  (the second, stricter filter)
--------------------------------------------------------------------------------
Model: claude-haiku-4-5

For each new (deduped) article, Claude is given the title, snippet, and source,
and must FIRST answer three yes/no questions BEFORE categorizing:

  Q1. Does the article primarily discuss drones, UAS, or UAV systems?  (yes/no)
  Q2. Does the article involve at least one Latin American country?    (yes/no)
  Q3. Is the drone/UAS content the MAIN topic, not a passing mention?  (yes/no)

RULE: If ANY answer is "no", relevance_score is forced to 0.
      This is hard-enforced in code (lib/categorize.js) — not merely requested
      of the model — so a "no" on any question cannot be overridden.

Claude returns JSON:
  {
    "q1_about_drones":      "yes" | "no",
    "q2_latam_country":     "yes" | "no",
    "q3_main_topic":        "yes" | "no",
    "primary_category":     one of the 8 categories below,
    "confidence_suggestion":"high" | "medium" | "low",
    "confidence_reasoning": "...",
    "relevance_score":      0..10,
    "key_actors":           [...],
    "countries":            [...],
    "summary":              "one sentence"
  }


--------------------------------------------------------------------------------
6. ALLOWED CATEGORIES  (8 — Latin-American drone activity only)
--------------------------------------------------------------------------------
  offensive_use        attacks, strikes, criminal use
  defensive_systems    C-UAS, interception, jamming
  military_investment  state procurement, budgets
  private_investment   company funding, R&D
  new_technology       new platforms, innovations
  regulatory           laws, agreements, policy
  criminal_vnsa        cartel, gang, VNSA activity
  diplomatic           foreign suppliers, transfers

If an article doesn't fit any of these for LATAM drone activity, its
relevance_score is pushed below the keep-threshold (treated as off-topic).


--------------------------------------------------------------------------------
7. KEEP / DISCARD THRESHOLD & RESULTING STATUS
--------------------------------------------------------------------------------
RELEVANCE_THRESHOLD = 5

  relevance_score >= 5  -> KEEP, insert with status = 'new'
  relevance_score <  5  -> DISCARD (never inserted)
  Claude call fails      -> insert with status = 'uncategorized' (manual review;
                            never silently dropped)

Cleanup pass (cleanup-irrelevant.js) re-checks already-stored rows through the
SAME gate and marks off-topic ones:
  status = 'rejected'
  notes  = 'Rejected: not relevant to LATAM drone topic'
(Articles already imported into the Hub — imported_to_hub = true — are skipped.)

Article status values overall:
  new | uncategorized | pending_review | approved | rejected | published
