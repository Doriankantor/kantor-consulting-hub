───────────────────────────────────────────────────────────────────────────────
1. WHAT THE APP IS
───────────────────────────────────────────────────────────────────────────────
"Project management for journalists and analysts." A cross-platform desktop app
(macOS + Windows) that combines:
  • A Kanban/list/timeline/calendar project workspace (Trello-style boards)
  • Team collaboration: inbox, chat, @mentions, notifications, assignments
  • A CRM (clients + contacts) and analytics dashboard
  • Google integration: Calendar sync, Drive backup/browser, Gmail invites
  • A "Source Intelligence" + "Info Pages" pipeline that auto-pulls news, runs
    it through a Claude relevance gate, and one-click publishes to GitHub repos
  • Per-user accounts with Supabase Auth, admin roles, and a built-in auto-updater

Built originally for a "Contested Skies" (drone / LATAM) news-monitoring use case,
plus a "Trump Immigration" visual project, with reusable pipelines for future ones.

3. ARCHITECTURE
───────────────────────────────────────────────────────────────────────────────
  Electron three-process model:
    • Main process    src/main/index.ts        app lifecycle, window, updater
                      src/main/db.ts            SQLite schema + migrations (987 ln)
                      src/main/ipc/index.ts     201 IPC handlers (3,731 ln)
                      src/main/google/drive.ts  DriveSync class (hub OAuth backup)
                      src/main/google/userGoogle.ts  per-user Google OAuth
                      src/main/google/gmail.ts  invite emails
    • Preload         src/preload/index.ts      exposes window.api.* (320 ln)
    • Renderer        src/renderer/src/...      React app (pages + components)

  Data flow:   React → window.api.* (preload) → ipcMain.handle → SQLite/Supabase/Google
  Local DB is the source of truth; Supabase handles cross-device auth + Drive
  handles backup. Env vars use the MAIN_VITE_ prefix (Supabase URL/key, Google
  client id/secret).

  Renderer contexts:
    AuthContext        login state, roles, first-login flow
    WorkspaceContext   boards/tasks state + local SQLite persistence
    ThemeContext       light/dark + theme presets
    UpdateContext      auto-update checks, banner, 8h interval / 24h dismiss

8. BUILD, RELEASE & DISTRIBUTION
───────────────────────────────────────────────────────────────────────────────
  npm run dev          electron-vite dev (hot reload)
  npm run build        compile only (no package)
  npm run make:mac     build universal macOS DMG locally
  npm run release      build + publish mac (dmg+zip, universal) AND win (nsis x64)
                       to GitHub Releases, then prune old local artifacts
  publish.command      double-click release from Finder
  install.sh           one-command install that bypasses macOS Gatekeeper

  Auto-update: electron-updater checks GitHub Releases every 8h; downloaded
  updates strip the macOS quarantine flag and install via a Terminal .command
  script (friendlier than the silent electron-updater install). Manual "Check
  for updates" is visible to all users; banner can be dismissed for 24h.

  iCloud setup: project lives in iCloud Drive. node_modules/dist/out are symlinks
  to *.nosync folders so iCloud skips them. Run `bash setup-new-device.sh` once
  on a new Mac after iCloud syncs the folder.
