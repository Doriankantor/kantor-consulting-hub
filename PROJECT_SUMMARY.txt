1. WHAT THE APP IS
───────────────────────────────────────────────────────────────────────────────
"Project management for journalists and analysts." A cross-platform desktop app
(macOS + Windows) that combines:
  • A Kanban/list/timeline/calendar project workspace (Trello-style boards)
  • Team collaboration: inbox, chat, @mentions, notifications, assignments
  • A CRM (clients + contacts) and analytics dashboard
  • Google integration: Calendar sync, Drive backup/browser, Gmail invites
  • A "Source Intelligence" + "Info Pages" pipeline that auto-pulls news, runs
    it through a Claude relevance gate, and one-click publishes to GitHub repos
  • Per-user accounts with Supabase Auth, admin roles, and a built-in auto-updater

Built originally for a "Contested Skies" (drone / LATAM) news-monitoring use case,
plus a "Trump Immigration" visual project, with reusable pipelines for future ones.

DESIGN PRINCIPLES / CONVENTIONS
───────────────────────────────────────────────────────────────────────────────
  • Realtime convention. Any feature backed by SHARED, CLOUD-STORED data must
    be registered as a consumer of the realtime manager
    (src/main/cloud/realtimeManager.ts) so changes propagate live across
    machines — the "plug in as config, not new mechanism" pattern; boards
    (boardsRealtime.ts) is the reference implementation. Full cost per shared
    feature: cloud table + RLS + main-process enforcement + realtime
    registration + one-time SQL (publication + replica identity) +
    two-machine verify. LOCAL/per-device data (e.g. notifications today) is
    exempt — nothing to propagate. When adding a feature, decide first whether
    its data is shared; if yes, realtime is part of "done", not optional.
  • Realtime DB setup requires an app RESTART to take effect. Adding tables to
    the supabase_realtime publication only affects sockets that subscribe
    AFTER the change. A running app subscribed before the tables were added
    will NOT receive their events until fully quit and relaunched (not just
    reloaded). Symptom if missed: writes land and navigate-away/back shows
    them, but nothing updates live. Restart both machines after a publication
    change before testing.
  • Database schema is documented in /sql (catalog baseline + realtime config),
    forward-from-today. When you run schema SQL in the Supabase editor, also
    save it as a dated file in /sql and commit it.
